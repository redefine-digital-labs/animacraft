Rules

Selection logic

cannot combine with
Part rules Prevent two whole part groups from being selected together, such as two incompatible hairstyles.
Item rules Prevent specific items from being selected together, such as one hat clipping through one hairstyle.
Hair: Twin Tails cannot combine with Hat: Tall Crown

Rule Scope

What becomes enforceable

Draft: creator tests visual compatibility locally. Template publish: allowed rule schema is stored on Sui. Soul authorization: the recipe must satisfy the published rules. Trade: license and royalty rules travel with the OC.

    On-chain rules for OC creation
    
            
    
              Animacraft keeps the maker experience simple while Sui and Walrus record Maker provenance,
              license policy, OC ownership, and the exact recipe used to create each character.

    No backend signer
    
            
    Creators and users sign their own transactions. Public reads use Sui GraphQL, while every core write belongs to Sui objects and wallet PTBs.
    
          
    
          
    
            W
            
    Walrus as asset layer
    
            
    Item PNGs, optional picker icons, Maker manifests, finished OC images, and profile JSON resolve from a certified Quilt Blob ID plus file identifier.
    
          
    
          
    
            PR
            
    Open-source releases
    
            
    Production changes land through pull requests, CI checks, and CODEOWNERS review before shipping to Vercel.
    
          
    
        
    
        
    
          
    
            
    Production Architecture
    
            
    Creator to player without a backend database
    
            
    Wallets sign core writes, Walrus stores assets, and Sui verifies ownership and recipes. Published Maker events drive the public gallery without an Animacraft database.
    
          
    
          
    
        
    
        
    
          
    
            
    Creator Guide
    Build one Character Maker
    
            Part → Item → Image
          
    
          
    
            
    01Library & Maker Top
    Create or open a Maker in the library, then use Maker Top to review structure, files, rules, and release readiness.
    
            
    02Character Maker
    Add a Part, choose Standard, Left-right pair, or required Last Bastion, then define Layers, Colors, Items, icons, and each PNG image cell.
    
            
    03Composition Order
    Arrange layers across Parts while the selected Item images remain visible on the same live canvas.
    
            
    04Rules & Palette
    Prevent incompatible choices and link color controls across related Parts.
    
            
    05Preview Check
    Validate visible Parts, rule references, and every required public Item × Layer × Color image cell.
    
            
    06On-chain Publish
    Prepare one Walrus quilt, register and upload it, certify availability, then publish and share the OCMaker on Sui. Interrupted stages can resume locally.
    
          
    
        
    
        
    
          
    Maker Data Model
    Where each setting belongs
    
          
    
            
    PartName, immutable type, menu icon, visibility, remove/default behavior, owned Layers, Colors, and Items.
    
            
    LayerOwned by one Part. Stores name, global composition position, offset, opacity, and blend mode.
    
            
    ColorOwned by one Part. Expands the image matrix and may link to colors in other Parts.
    
            
    ItemOne user-selectable option inside a Part, with display order, visibility, picker icon, and item rules.
    
            
    Item imageOne PNG for a specific Item × Layer × Color cell. These files become Walrus quilt patches.
    
            
    OCMakerThe shared Sui object binds original creator, Quilt Blob ID, public Parts/Items/Colors, rules, mint economics, license policy, and archive state.
    
            
    MakerTreasuryA shared coin-typed vault receives exact native-USDC mint fees and records collection and withdrawal totals.
    
            
    MakerAdminCapThe transferable management right controls economics, withdrawals, and lifecycle. Cap ownership is separate from original art provenance.
    
          
    
        
    
        
    
          
    Maker Lifecycle
    Delete locally, archive on-chain
    
          
    
            
    Local draftIndexedDB stores wallet-scoped structure, source files, and upload checkpoints. Maker, Part, Item, optional Layer, and extra Color records may be permanently deleted before publication.
    
            
    Published MakerPublication locks art, rules, and the certified Walrus manifest. The Cap holder may only change future mint economics and archive state.
    
            
    Archived MakerThe creator may archive or restore a published Maker with a wallet signature. Archive blocks new Soul authorizations while preserving existing Souls, rights snapshots, and provenance.
    
            
    Walrus retentionProduction uploads request 53 Mainnet epochs, currently about two years. Removing a local reference does not erase certified storage, but retention must be extended before expiry.
