Parts

Parts are the creator-facing asset groups users choose from when making an OC. Each part can contain multiple PNG items, linked colors, rules, and license metadata.

Standard parts Normal selectable asset groups. Use this for hair, eyes, outfits, mouths, accessories, and backgrounds.
Left-right paired parts Mirrored or paired assets such as earrings, eyes, wings, or sleeves. Distance and offset can be adjusted together.
Last bastion parts Fallback items that remain selectable when rules would otherwise hide every option in a group.

Part Settings

Controls per part

Part name and internal creator label Type lock: standard, left-right pair, or last bastion cannot be changed after creation Part icon for the user menu, or hidden menu mode for fixed background parts Canvas anchor: upper-left X/Y, plus right-layer X for paired parts Remove button and default item state Allowed controls: move X/Y, scale, rotate, paired spacing

Item Settings

Items inside a part

No.1 item is the required base item and cannot be deleted or made private Item number supports bulk upload and can differ from display order Public/private status decides whether the item appears to makers Each item can have PNG layer files and a separate selection icon Display order controls the user-facing item picker

Rules

Selection logic

Part rules Prevent two whole part groups from being selected together, such as two incompatible hairstyles.
Item rules Prevent specific items from being selected together, such as one hat clipping through one hairstyle.
Hair: Twin Tails cannot combine with Hat: Tall Crown

Rule Scope

What becomes enforceable

Draft: creator tests visual compatibility locally. Template publish: allowed rule schema is stored on Sui. OC mint: recipe must satisfy the published rules. Trade: license and royalty rules travel with the OC.

Last Bastion

Fallback safety

Dress selected deselects Tops + Bottoms

If a rule would leave a required body area empty, a last-bastion part becomes the fallback. Animacraft should store this as a rule-bound required fallback object on-chain.

Color Linking

Linked palettes

Connect color-controlled items so a user can change one palette and keep related layers synchronized, such as front hair, back hair, eyebrows, and shadows.

Preview Validation

Publish readiness

  • Standard or left-right paired parts are registered.
  • At least one part is visible in the user menu.
  • One or more item images still need final PNG files before release.
  • Rules should be tested against clipping and empty-state combinations.

User Flow

What makers will see

Template opens as a clean OC creation page. Only visible parts appear in the user menu. Invalid combinations are disabled before minting. Preview exports a recipe, cover image, and metadata package.

Validation Matrix

Required filesEvery public item needs item images for every required layer.

On-chain rules for OC creation

            

              Animacraft keeps the maker experience simple, while the protocol records material provenance,
              template licenses, OC ownership, trade settlement, and remix relationships.
            

          

        

        

          

            0x
            
No backend signer

            
Creators and users sign their own transactions. Animacraft can use optional indexers later, but core writes belong to Sui objects and wallet PTBs.

          

          

            W
            
Walrus as asset layer

            
All PNG layers, cover sheets, manifests, rendered OCs, and profile JSON should resolve from Walrus blob ids.

          

          

            PR
            
Open-source releases

            
Production changes land through pull requests, CI checks, and CODEOWNERS review before shipping to Vercel.

          

        

        

          

            
Production Loop

            
Creator to player without a backend database

            
Animacraft should work like a browser-native on-chain app: wallet signs, Walrus stores, Sui verifies, and events become the index surface.
